## PART 3: VARIATION STRATEGY FOR REMAINING 110 FILES

The remaining 110 files are generated using the same templates with systematic variations:

### PDF Runbooks (30 remaining files)
- **Template**: Same structure as Document 1 (payment_processing_runbook)
- **Variations**:
  - Different regions: us, uk, ca, de, in
  - Different environments: prod, staging, dev
  - Different runbook types: disbursement, repayment, document_verification, eligibility_check, credit_assessment
  - Different error codes: ERR_PAY_LOAN_001 through ERR_PAY_LOAN_010
  - Different payment gateways: Razorpay, Stripe, PayPal, local gateways
  - Different incident dates and reference numbers

### Word Documents (20 remaining files)
- **Template**: Same structure as Document 2 (loan_application_faq)
- **Variations**:
  - Different document types: FAQ, SLA policy, escalation guide, onboarding, repayment guide
  - Different domains: payment_processing, authentication, redis_cache, kafka_queues, database, notifications
  - Different question sets: 25+ unique FAQs per domain
  - Different SLA metrics: Response times, resolution times, availability targets
  - Different escalation procedures: Based on severity levels

### TXT Files (30 remaining files)
- **Error Code Files**: Same structure with domain-specific error codes
  - Payment: ERR_PAY_LOAN_001-010
  - Loan Application: ERR_APP_LOAN_001-010
  - Authentication: ERR_AUTH_LOAN_001-010
  - Redis: ERR_REDIS_LOAN_001-010
  - Kafka: ERR_KAFKA_LOAN_001-010
  - Database: ERR_DB_LOAN_001-010
  - Notifications: ERR_NOTIF_LOAN_001-010

- **Config Reference Files**: Domain-specific configuration parameters
  - Payment: Gateway settings, retry policies, timeout values
  - Loan Application: Eligibility thresholds, document requirements
  - Authentication: Session timeouts, lockout policies, OAuth settings
  - Redis: Cache TTL, memory limits, eviction policies
  - Kafka: Consumer groups, topic configurations, retention policies
  - Database: Connection pools, query timeouts, replication settings
  - Notifications: Rate limits, provider settings, delivery retries

### Image Files (9 remaining files)
- **Template**: Architecture diagrams with OCR text
- **Variations**:
  - Different system components: Payment flow, loan application flow, authentication flow
  - Different diagram types: Sequence diagrams, component diagrams, data flow diagrams
  - Different regions: US, UK, Canada, Germany, India specific architectures
  - OCR text variations: Component names, flow descriptions, error handling paths

## CONTENT GENERATION NOTES

1. **All documents are 5-6 pages** (~2,500-3,000 words) except TXT files (1-2 pages)
2. **Content is strictly customer-support oriented** - no general knowledge or creative content
3. **Templates ensure consistency** while variations provide diversity
4. **Error codes are domain-specific** and follow naming conventions
5. **Past incidents are realistic** with dates, root causes, and resolutions
6. **OCR text for images** is provided separately for RAG indexing
